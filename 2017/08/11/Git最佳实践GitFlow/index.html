Git最佳实践GitFlow

发表于 2017-08-11 | 分类于 编程 | 阅读次数:

GitFlow是现在比较流行的git分支模型上一张神图

具体的操作步骤如下：

  1. 主分支为master,这个分支最近发布到生产环境的代码，最近发布的Release， 这个分支只能从其他分支合并，不能在这个分支直接修改。发布分支为tag,每个tag为一个发布版本。

  2. develop分支是开发分支，这个主要合并与其他分支,所有也是合并分支，所有代码的合并应该在develop完成

  3. Feature分支从develop创建，用于开发新的功能分支。

  4. Feature分支功能开发完成,提交合并分支到develop

  5. Release待发布分支，一旦打了Release分支之后不要从Develop分支上合并新的改动到Release分支。基于Develop创建release分支。

  6. 完成release

  7. 发布新版本Tag,从master创建tag，tag为线上版本

  8. hotfix分支，生成环境问题修复。

  9. 完成hotfix

当然讨厌麻烦命令的可以看看git-flow工具，很方便实践gitflow。
