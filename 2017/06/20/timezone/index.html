MySQL Time Zone 时区问题

发表于 2017-06-20 | 分类于 编程 | 阅读次数:

MySQL的Time Zone很有意思，也很容易被忽略。

  1. 时区的更改会影响MySQL时间函数的值，例如NOW(),CURDATE()等
  2. 时区的更改会影响timestamp类型列的值，注意TIMESTAMP列的值从当前时区转换为UTC用于存储，当进行检索的时候从UTC转换到当前时区，所以当时区（或者会话时区）发生变化时，timestamp的值不同。
  3. 时区影响FROM_UNIXTIME(),UNIX_TIMESTAMP()的时间转换值。
  4. 除系统时区影响上述，当修改当前会话的时区时也影响上述。

MySQL Server时区详解

查看数据库当前的时区设置：

  • system_time_zone. 当前MySQL Server 启动的时候，会尝试确定当前主机的时区，并使用它来设置system_time_zone, 这个的不会改变.

    您可以在启动时使用 --timezone = timezone_name选项为mysqld_safe设置MySQL Server的系统时区。您也可以在启动Mysqld之前设置TZ环境变量来进行设置。 --timezone或TZ的允许值取决于系统。

  • time_zone. 全局time_zone系统变量表示服务器当前正在运行的时区。time_zone的初始值为“SYSTEM”，表示服务器时区与系统时区相同。

    1. 在启动时可以使用命令行上的--default-time-zone = timezone选项显式指定初始全局服务器时区值，也可以在选项文件中使用以下行：
      default-time-zone='timezone'
      例如：
      可以通过修改my.cnf
      在 [mysqld] 之下加
      default-time-zone = '+8:00'

    2. 如果您具有权限，则可以使用SQL语句在运行时设置全局服务器时区值：
      mysql> SET GLOBAL time_zone = timezone;

    3. 当前会话时区，接的每个客户端都有自己的时区设置，由会话time_zone变量给出。最初，会话变量从全局time_zone变量获取其值，但客户端可以使用此语句更改其自己的时区：
      mysql> SET time_zone = timezone;

      当前会话时区设置会影响到对区域敏感的是时间值的显示和存储。
      影响有：

      • NOW() 或者 CURTIME()等函数；
      • 类型为TIMESTAMP的列的值( TIMESTAMP列的值从当前时区转换为UTC，用于存储，并从UTC转换到当前时区进行检索)。
      • 当前时区设置不会影响由UTC_TIMESTAMP（）或DATE，TIME或DATETIME列中的值显示的值。这些数据类型中的值也不是存储在UTC中;仅当从TIMESTAMP值转换时，时区才适用于它们。如果要对DATE，TIME或DATETIME值进行特定于语言环境的算术，请将其转换为UTC，执行算术，然后进行转换。

    4. 查看全局时区，以及当前会话时区
      mysql> SELECT @@global.time_zone, @@session.time_zone;

    5. time_zone的值。

      • SYSTEM 表示时区与系统时区相同。
      • 该值可以作为指示与UTC的偏移量的字符串给出，例如“+10:00”或“-6:00”。中国为“+08:00”
      • 时区的命名 ‘Europe_Helsinki’, ‘US_Eastern’, or ‘MET’. 只有在创建并填充mysql数据库中的时区信息表时，才能使用命名时区。
